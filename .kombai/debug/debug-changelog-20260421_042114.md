## Changes Made

### 1. Fixed Foreign Key Constraint Name Collision
- **File:** `controllers/TableBuilderController.php`
- **Change:** Modified `buildForeignKeyConstraintName` to append a short 4-character unique hash to constraint names.
- **Reason:** In MySQL, renaming a table (during sync/rebuild) keeps the original foreign key names. When the tool tried to create a new version of the table with the same name and the same "clean" constraint names, it failed with a "Duplicate foreign key constraint name" error. The unique hash ensures the names are unique even during the temporary coexistence of the old (backup) and new table.
- **Revert:** Remove the line that appends the hash in `buildForeignKeyConstraintName`.

### 2. Added Foreign Key Check Suppression during Table Sync
- **File:** `controllers/TableBuilderController.php`
- **Change:** Added `SET FOREIGN_KEY_CHECKS = 0` and `SET FOREIGN_KEY_CHECKS = 1` around the table replacement logic in `syncUpdatedPhysicalTable` and `actionExecuteSql`.
- **Reason:** This prevents parent-child relationship constraints from blocking the table rebuild process (e.g., when a table is referenced by other tables).
- **Revert:** Remove the `SET FOREIGN_KEY_CHECKS` commands and the `try-finally` blocks.
