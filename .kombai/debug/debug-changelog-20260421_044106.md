## Changes Made

### 1. Fixed "Unknown Property" in Form Submission
- **File:** `models/FormSubmission.php`
- **Change:** Added `firebase_uid`, `firebase_email`, and `firebase_name` as public properties/rules.
- **Reason:** These columns exist in the local database but were not declared in the model, causing Yii to throw an "Unknown Property" error when trying to assign values to them during submission.
- **Revert:** Remove the property declarations and rules for these 3 fields.

### 2. Fixed Database Connection for Custom Table Insertion
- **File:** `controllers/FormController.php`
- **Change:** Modified `persistSubmissionToCustomTable` to use the result of `getPhysicalDb()` instead of `Yii::$app->db`.
- **Reason:** In project-based contexts, the project database is often different from the main application database. Using `Yii::$app->db` directly would fail if the target table only exists in the project-specific database.
- **Revert:** Change `$targetDb` back to `Yii::$app->db` in that method.

### 3. Hardened Schema Extraction
- **Files:** `models/Form.php`, `controllers/FormController.php`
- **Change:** 
    - In `models/Form.php`, ensured `getSchema()` uses the correct storage column (legacy `schema_json` or new `schema_js`).
    - In `controllers/FormController.php`, added checks to ensure schema blocks are arrays before processing.
- **Reason:** Variations in how schema data is stored (legacy vs new) or malformed JSON could cause runtime errors.
- **Revert:** Restore the original simpler schema extraction logic.
